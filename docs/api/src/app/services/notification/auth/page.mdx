# Authentication

The Auth API allows you to authenticate users and send add firebase token.

---

## Identify User {{ tag: 'POST', label: '/v1/notifications/user/identify' }}

<Row>
  <Col>
    You can register user from your system to the notification service.
    ### Required attributes

    <Properties>
      <Property name="user_id" type="string">
        The user id in your system.
      </Property>
      <Property name="email" type="string">
        The user email.
      </Property>
      <Property name="phone_number" type="string">
        The user phone number.
      </Property>
    </Properties>
  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/v1/notifications/users/identify">

      ```json {{title: 'JSON'}}
      {
        "user_id": "11",
        "email": "aungmyatmoe@breeze.com",
        "phone_number": "09790000000"
      }
      ```


    </CodeGroup>

    ```json {{title: 'Response'}}
    {
      "message": "User identified successfully",
    }
    ```

  </Col>
</Row>

---

## Register Push Token {{ tag: 'POST', label: '/v1/notifications/token' }}

<Row>
  <Col>

    This endpoint allows you to register a user's Firebase token to the notification service.

    ### Required attributes

    <Properties>
      <Property name="user_id" type="string">
        The user id in your system.
      </Property>
      <Property name="type" type="string">
        The type of the token. It can be `FCM` or `APN`.
      </Property>
      <Property name="token" type="string">
        Firebase (FCM) or Apple (APN) token.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/v1/notifications/token">

      ```json
      {
        "user_id": "11",
        "type": "FCM",
        "token": "dPR..." // Firebase token
      }
      ```

    </CodeGroup>

    ```json {{title: 'Response'}}
    {
      "message": "Token added successfully",
    }
    ```

  </Col>
</Row>
